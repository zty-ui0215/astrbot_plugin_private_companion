# 更新日志

## 3.8.1

- 优化主动消息和主动语音生成：继续走 AstrBot 主 Agent / 会话历史链路，同时按私聊会话串行化主动与被动 LLM 请求，降低 sqlite `database is locked` 冲突。

## 3.8.0

- 新增私聊用户关系角色边界：朋友用户不再获得窥屏、独立生图、JM/本子推荐、私密阅读分享、群聊隐私转述等敏感主动能力。
- 私聊回复注入增加朋友保护：朋友聊天不再注入书架夹层密码、秘密书架阅读记录、私密阅读偏好或 JM 阅读上下文。
- 优化陪伴面板私聊用户详情：用户级配置只保留每日主动限额，并展示对应关系网用户词条。
- 主动候选记录支持按私聊用户筛选，便于查看不同对象的主动候选来源和拦截结果。
- 修复群聊人格降噪开关识别：不再把它混作群上下文注入的关联参数，且运行时可独立于群上下文注入生效。

## 3.7.1

- 修复 v3.6.2/3.7.0 版本残留导致的私聊处理异常：主类补齐用户休息静默兜底方法，避免 `_apply_user_rest_silence_from_message` 缺失时中断私聊回复。
- 修复插件注册版本号仍显示为 `3.6.2` 的问题，统一插件内部注册版本、元数据和面板文档版本。
- 优化非目标私聊身份防串：非主用户私聊时先防止套用主人/恋人专属设定，再按当前 QQ 精确注入低风险关系网资料。
- 关系网用户词条新增性别字段，支持面板编辑、保存、导入和提示词注入。
- 新增私聊转述回执：`pc_relay_message` 私聊询问可设置等待对方回复并自动把结果带回发起会话，可选回报前向对方二次确认。
- 修复时间感知可能跟系统时区跑偏的问题：今日日期、当前时段、主动调度、日程细化、休息静默与合并消息时间展示统一优先使用 `environment_perception_timezone`。

## 3.7.0

- 优化拟人化连续性：主动行为会记录更细的行动后果，后续私聊回复可引用最近行为反馈，减少状态割裂。
- 优化主动消息意图：降低泛泛问候和空泛插话倾向，让主动理由更少、更具体，并根据近期被忽略情况收敛表达。
- 优化语言风格疲劳：主动表达会避免连续复用同类句式、语气和开场，让回复变化更自然。
- 新增群聊人格降噪提示：群聊场景会弱化过度亲密、状态自报和私聊式撒娇，减少硬插话感。
- 合入 QQ 空间 Cookie / g_tk 一致性修复：同一次查询、发布、点赞、评论会复用同一份 Cookie 与对应 g_tk，避免不同 QZone 端点 Cookie 不一致导致鉴权失败。

## 3.6.2

- 修复直播自动回应默认使用 Bot 自己私聊会话后，Bot 自己 QQ 可能被陪伴插件误当作主动目标的问题；私聊回流事件会忽略 sender/self 相同的 Bot 自发消息，主动对象解析也会排除 Bot 自己 QQ。
- 优化 B 站直播观众与关系网联动：关系节点支持 `bili:UID` / `bili_live_xxx` 外部身份键，外部节点可在面板中编辑并绑定到已有 QQ 节点。
- 绑定 B 站外部身份到 QQ 节点时，会合并名称、别名、说明、重要记忆和观众活跃画像，后续直播弹幕优先识别到绑定后的 QQ 关系节点。
- 优化关系网页面：外部身份节点会显示绑定状态和外部身份数量，新增节点表单允许 QQ 号或 B 站外部身份键。

## 3.6.1

- 新增用户休息静默锁：用户明确表示晚安、睡觉、休息、午休、别打扰或明天再说后，普通主动消息会暂停到合理时间；用户说醒了、起床了或可以聊了会解除静默。
- 优化群聊 @ 休息用户场景：群友 @ 正在休息的目标用户时，不私聊、不再次 @ 目标用户，改为走群聊 LLM 自然提醒发起者晚点再叫，并加 10 分钟防刷。
- 优化跨会话转述边界：私聊转述或群里继续 @ 休息中的目标用户会被拦截，延迟转述仍可保留到目标用户出现后再发送。
- 修复 TTS 与表情包/图片混合消息链兼容：混入非文本组件时仍会处理 `<tts>`，不再把标签原样发出。
- 优化 TTS 语音发送顺序：按 LLM 输出顺序拆分发送文本与语音，避免 NapCat/QQ 在同一条混合链中把语音泡泡渲染到文字前面。
- 修复 QQ 空间自动生活说说可能泄漏内部状态变量的问题；自动草稿会过滤“心理能量”“/100”等内部汇报词。
- 优化被动分段清理：新增“仅句尾清理”范围，减少正文标点或空格被全段误删。

## 3.5.2

- 适配群聊引用 Bilibili 动态/合并转发卡片：支持 OneBot `multimsg`/`resid` 展开、卡片文本、链接和本地渲染图提取，引用 B 站插件发送的合并消息时也能进入图片视觉链路。
- 修复群聊引用合并消息被带歪的问题：合并消息与图片转述提示会保留作品名、活动名和日期，不再用外部搜索或同系列游戏信息替换当前图片内容。
- 增强转发节点身份判断：合并消息里由 Bot 自己发送的节点会标记为 Bot 自己，避免转到私聊后只认出内容却认不出来源。
- 修复 AstrBot 上下文压缩遇到 `TextPart`/图片消息段时可能报 “not JSON serializable” 的问题，并将主动消息与 Bilibili 摘要入库内容改为纯文本兼容格式。
- 优化图片视觉缓存命中率：新增跨 provider 命中回退、重复缓存变体清理和偏向保留高命中条目的淘汰策略。
- 优化 QQ 自定义状态同步：启动后会补一次当前状态同步，同一状态缓存超过 10 分钟后允许重新下发，减少重启或协议端掉状态后仍误判已同步的情况。

## 3.5.1

- 增强撤回消息匹配：普通消息缓存会同时登记 OneBot 原始消息 ID 和 AstrBot 包装消息 ID，避免撤回通知与缓存 ID 不一致导致“没有可转述的撤回消息”。
- 新增自然语言查询撤回消息：授权用户可直接问“刚才撤回了什么”“谁撤回了，发的啥”等，Bot 会在当前会话内自然转述最近撤回内容。
- 新增 `陪伴 TTS语种 日语|中文|英语|默认` 管理命令，可运行时切换 TTS 语音语种并保存覆盖设置。
- 新增 TTS 本机播放与直播字幕联动开关：TTS 音频生成后可在运行 AstrBot 的电脑上播放，并可同步推送朗读文本到 `astrbot_plugin_live_stream_companion` 打字机字幕 overlay。
- 新增预约静默窗口：已有明确自预约/定时主动时，默认到点前 20 分钟暂停普通主动、链式追问和未回复补一句；若预约文本带有休息/睡觉/起床语义，会从预约创建起静默到到点，静默期间被压住的主动念头会暂存并在到点消息里顺带接上。
- 优化创作分享发送：`creative_share` 主动分享自动豁免主动分段，小说片段、诗和随笔等创作内容会尽量保持完整阅读节奏。
- 收紧撤回消息查询权限：自然语言查询沿用命令权限，私聊仅允许 Bot 管理员或配置目标用户，群聊额外允许群管理员；无权限时不会暴露或猜测撤回内容。
- 同步远端 README 最新能力边界说明，并更新插件版本到 `3.5.1`。
