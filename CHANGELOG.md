# 更新日志

## 5.4.1

- 更新版本号至 `5.4.1`，聚焦修复人设参考图路径配置同步问题。
- 修复人设参考图路径保存后看起来没变：配置写入现在优先同步官方配置页分组项，清空参考图路径也会同步到运行态，避免旧平铺兼容值或内存旧值继续显示/生效。
- 修复撤回增强误判戳一戳事件：发送前取消回复只对普通消息事件的触发 ID 做 `get_msg` 存在性检查，`notice/poke` 这类非消息事件不再因平台返回“消息不存在”被记录为已撤回并取消发送。

## 5.4.0

- 更新版本号至 `5.4.0`，整理 `5.3.7` 之后的在线生图平台适配、私聊放行修复、备份导入兼容和本地兜底收束。
- 新增 `在线生图模型平台` 配置项，可选 `auto / openai / bailian`；扩展页保存后会按平台走对应接口，不再把所有在线生图都强行当成 OpenAI 图片协议。
- 新增阿里云百炼 / DashScope 原生生图适配：支持按百炼平台提交生图请求，并根据模型能力自动在多模态同步返回与异步任务轮询之间切换。
- 保留原有 OpenAI 兼容在线生图链路；已有 `external` 后端配置无需迁移，只需补充平台项即可。
- 在线生图地址增加自动修复：共享域名、专属域名、`/api/v1` 根地址、完整接口地址，以及常见百炼控制台页面链接都会尽量自动归一化为实际可请求的地址，并在生图排障摘要里显示修正后的结果。
- 修复非目标/未启用私聊被插件误吞的问题：未开启“仅保留主动能力”时，非陪伴私聊会放行给 AstrBot 默认主链或其他插件处理，仅保留请求阶段的轻量身份防串提示。
- 调整“仅保留主动能力”文案：替换旧的“关闭被动能力”显示名，并明确它只跳过本插件的普通被动增强，不会阻止默认回复或其他插件。
- 修复配置备份导入兼容性：导入端可识别 `{package}`、`{data}` 等外层包装，并兼容旧备份的 checksum 计算顺序，避免刚导出的备份预览时报“备份校验失败”。
- 收束本地兜底外发：被动复读不再用固定短句替换；过长闲聊只从模型原文里截取自然句；单图与空间动态在模型兜底失败时改为跳过并记录日志，避免把硬编码文案发给用户。
- 调整自登记拦截默认回复：旧默认“你是小猪”会迁移为更普通的“这个称呼我不记。”，配置页保存/读取也同步兼容旧值。
- 收敛主动/群聊分享动机里的“闹”类说法：群里持续提到 Bot 时改用中性描述，避免生成“有人一直闹我”这类带抱怨感的表达。
- 移除主动消息风格兜底固定文案：不再按“温柔/活泼/校园风”随机抽取成品句子；兜底只选择主动原因和内部意图，最终正文仍走框架生成与发送前审核。“陪伴 语气”改为自由记录简短语气偏好。
- 优化日程 `message_seed` 留白逻辑：没有自然开口时要求直接留空，不再生成“这段没什么想说的”“先不吵你”“脑子空空的”这类占位句；旧日程里的同类占位种子也会清理为空。
- 收束“空一下/空下来”类内部措辞：下午空闲候选改为“短暂休息/节奏缓下来”，并清理“下午空一下”这类被模型压缩出的异常 `message_seed`。
- 修复排障页最近注入列表预览污染：TTS 规则、请求提示词等内部模块正文不再被当作“最近回复消息”标题；旧记录读取时也会过滤这类内部提示，并在前端限制为短预览。
- 继续补强 Issue #39 的备份导入兼容：checksum 校验会兼容旧导出在 `checksum_algorithm`、JSON 空格和 key 排序上的历史差异；前端导入也会先解开 `{success,data}`、`{package}` 等外层响应包。
- 排障页底部新增常见问题区：整理“为什么没有主动消息”“为什么生图失败”“疑似注入带偏”“为什么没有注入记录”等排查路径，并提供跳转和链路测试按钮。
- 优化 QQ 空间 Cookie 获取：后台评论收件箱会兼容更多 OneBot 包装层和平台管理器字段；拿不到 Cookie 时只记录简短失败状态，不再刷出调度器 traceback。

## 5.3.7

- 更新版本号至 `5.3.7`，聚焦收束被动回复、生图预设、Token 概览和扩展页配置体验。
- 优化被动回复自检：短闲聊、天气短句或轻状态被主链扩写成关心清单、建议清单、天气小作文时，默认复核模式也会介入压短；若模型改写后仍过长，会保守兜底成一句自然接话。
- 优化主动生图提示词：参考通用生图插件的预设格式，新增生图场景预设；主动拍照、每日穿搭和自然语言生图会自动套用自拍、COS、穿搭、头像、房间日常、表情包等场景预设，并在排障记录里显示命中的预设。
- 修复扩展页 Token 概览误判：打开拓展页时 `/overview` 会带轻量 Token 摘要，首页和快捷卡片不再必须点进 Token 页后才显示消耗；Token 页仍会单独拉取完整明细。
- 增加火山方舟免费额度结束提醒：README 和模型页都会提示协作计划免费额度将在 `2026-06-30` 结束，方便用户及时控制成本。
- 补齐 Token 页任务名汉化：主动人格判定、单图回复主链、外界信息关联、QQ 空间、智能防抖、休息醒来、群聊续接等内部任务不再直接显示英文 key。
- 修复功能详情页参数容易丢草稿：在生图等功能详情里修改 ComfyUI 工作流后，点击“返回功能列表”或面包屑返回会先保存当前关联参数，避免退出再进看起来回退。
- 优化在线生图超时处理：在线图片 API 长时间无响应时会记录简短的超时原因并进入后端回退，不再把 aiohttp 内部 traceback 当成普通警告刷屏。
- 优化功能开关页布局：动态提示词注入位置单独放在第一行，关闭被动能力与主链会话锁兼容模式并列，减少卡片挤压。
- 调整“关闭被动能力”说明：弱化缓存用途，改为强调避免被动链路误接管、误识别；配置键保持兼容。
- 优化排障页注入检查：改为按最近 10 次回复消息聚合展示；点开单条消息即可查看该次回复对应的完整注入模块，不再只按 `request/passive/proactive/tts` 碎片分桶查看。
- 补强 AI 日报/早报问法：新增 `AI日报 / ai日报 / 日报 / AI早报 / ai早报 / 早报` 的命令入口；自然聊天里问到 AI 日报/早报时会优先读取 `ai_daily` 记录，并在记录不是当天时明确说明具体日期。
- 收紧排障页误警告：`用户在该问候窗口内已经活跃过`、`潜在念头窗口已过期`、`多来源合并`、`群聊分享候选已过期` 等正常候选过滤不再被当作警告原因；`photo_text` 候选也只在异常拦截时才提示。
- 调整环境感知文案：从“不要提到自己读取了环境”改为“用户明确问到时可自然回答，没问到时仅作为背景参考”，减少对正常回答的反向约束。
- 收束主动候选文案：移除“刚开机”这类系统感表达，并同步调整早安、午间、下午、睡前、天气、日常碎片、饥饿状态、动作后续和未回复窥屏等 scene/motive，减少“轻轻碰一下/递过去/落一下脚/偷偷看”这类容易外溢的抽象动机；发送前会额外清理“想和用户说一句”这类内部动机尾巴，避免写进正文。

## 5.3.4

- 更新版本号至 `5.3.4`，聚焦收束主动消息表达、群聊答疑误触和生图排障可观测性。
- 新增私聊主动发送目标映射：可配置 `大号QQ=小号QQ` 让主动消息、主动测试等发送到小号，同时继续用 `小号QQ=大号QQ` 做身份归并，保持大号用户档案和记忆不分裂。
- 优化旧配置兼容默认值：保留新版分组配置的同时补齐旧版扁平兼容键，减少启动时反复出现 `Config key missing` 和旧值被默认值遮蔽的问题。
- 优化主动消息生成与后处理：状态只作为语气底色，不再把洗漱、穿搭、天气、心情和日程逐项列成状态清单；发送前会收束“几个念头硬凑”的主动文本，只保留一个自然切口。
- 优化主动消息状态尾巴：减少“我刚才发呆/躺着/盯天花板”这类刻意状态汇报，同时保留真实生活片段。
- 优化主动消息近似去重：早晨天气问候会归一成稳定签名，避免“早上好刚醒看窗外没下雨”和“刚醒看窗外没下雨”这类换壳内容短时间内重复发送。
- 优化群聊答疑唤醒：开放疑问会参考接话场景、短反问/吐槽、附近已有讨论和问题目标，降低碰瓷式答疑误触。
- 新增群聊答疑发送前复核：`group_wakeup_question` 触发的群聊回复会在发送前由复核模型判断 `send/drop`，避免主链已生成后仍把碰瓷回复发出去，并加一次性标记避免反复调用。
- 优化排障中心生图信息：记录并展示最近真实生图调用的 trace、类型、后端、成败、参考图、路径、原因和提示词，生图测试结果也会显示本次测试 prompt。
- 修复生图排障记录缺少最近真实提示词的问题，普通主动拍照、每日穿搭、自然语言生图/改图和后端回退都会留下最近记录。

## 5.3.2

- 更新版本号至 `5.3.2`，聚焦修复自然语言生图误触、夹层密码管理和主动调度空转问题。
- 收紧自然语言生图触发：普通“做个插件/做个功能/生成日志”等不含图片目标词的句子不再误入生图额度判断；文生图需要明确出现图片、照片、插画、头像、壁纸、表情包或图等视觉目标词。
- 保留带图/引用图改图入口：携带或引用图片时，“改成红色基调”等视觉编辑请求仍可触发改图；未带参考图时不会误判。
- 人设参考图支持填写 http(s) 图片 URL：聊天指令和配置页都可使用，URL 会在首次使用时下载为本地稳定参考图并回写配置，避免每次自拍/人像生图都重复请求远程图片。
- 优化私聊单图识图延迟：新增单个识图模型超时控制，某个视觉 provider 503 或长时间重试时会快速临时降权并切换候选；单图等待超时后主链不再重复发起第二轮视觉转述，避免一张图拖到一两分钟才回复。
- 修复空私聊事件漏拦截：插件识别到平台空消息后会设置空结果并停止事件传播，避免默认 LLM 收到空 prompt 后误调用工具并把英文工具错误发到私聊。
- 收紧非目标私聊接管边界：未登记或未启用的私聊用户会在私聊消息入口停止默认主链传播，并在 LLM 请求入口兜底拦截，避免跳过陪伴注入后继续进入 AstrBot 默认 LLM，出现英文工具清单或 future_task 说明。
- 前置用户休息静默拦截：用户说“晚安/先睡了/别打扰”等明确休息信号时，在私聊消息入口保存静默状态后直接停止本轮传播，避免先跑记忆召回、情绪复核或主链 LLM 后再被睡眠闸门拦截。
- 补齐夹层密码注入日志：用户提到夹层、书柜、日记、密码、本子等相关内容但未注入时，会记录未注入原因、命中词、访问意图词和功能开关状态，便于排查是功能关闭、朋友用户、意图不足还是轻量被动链路跳过。
- 收束夹层密码触发边界：账号、登录、邮箱、Wi-Fi、API token 等外部凭据语境里的“密码/口令/钥匙”不再误触发书柜夹层密码注入。
- 优化夹层密码生成：恢复由模型按人格生成数字暗号，但不再参考重要日期；旧的 `1234`、重复数字、顺子等弱密码会自动轮换，模型输出不合格时回退到本地随机，并禁止回复时把密码解释成生日、纪念日或现实日期。
- 新增夹层密码管理指令：`陪伴 重置夹层密码` 可重新生成密码，`陪伴 输出夹层密码` / `陪伴 强制输出 夹层密码` 可由管理者直接查看真实密码。
- 夹层密码现在会保存一句生成理由；输出密码时会同步补齐理由，扩展页抽屉夹层密码栏旁会显示已保存的提示，方便用户知道密码为什么这么设。
- 补齐夹层密码注入外层日志：群聊、非目标私聊、被动注入关闭、休息闸门、主动-only 轻路径、用户未启用、marker 已存在、空注入等早退路径都会在命中夹层相关词时记录原因；明确夹层密码请求会退出轻量被动链路，避免短句跳过注入。
- 修复 Issue #37：关系/情绪闸门拦截主动消息时，`backoff` 会按 `backoff_until` 延后下一次调度；发送检查失败但没有未来调度时会兜底延后，避免主动调度器几秒一轮空转。
- 收紧群聊自然语言生图/改图入口：群聊里不再因为普通唤醒态、续接态或提到 Bot 名字而触发图生图；必须明确 @Bot 或回复 Bot 消息才会进入自然语言生图判断。
- 新增 `photo_generation_fixed_prompt` 配置项：所有生图提交后端前都会追加固定提示词，覆盖主动随手拍、每日穿搭、自然语言文生图和引用/携带图片改图。

## 5.3.0

- 更新版本号至 `5.3.0`，集中整理主动生图、每日穿搭、QQ 空间评论收件箱和身份隔离相关改动。
- 优化主动生图链路：在线 API 优先、失败回退到本地 ComfyUI；参考图支持本地配置和聊天指令/带图更新；排障测试收敛为自拍测试，避免误用普通文本模型请求图片接口。
- 新增每日穿搭能力：开启后在日程生成后按当天生活安排生成角色穿搭自拍，并替换扩展页左上角 logo；手动指令会先发送“等我换身衣服哦”，生成成功后发送“换好啦，你看”并携带真实图片。
- 优化每日穿搭提示词：强调角色脸部、头发、表情和服装完整可见，默认使用 1024x1024 正方形构图，并让扩展页 logo 使用完整显示，减少头像裁切和脸部缺失。
- 新增默认关闭的自然语言生图/改图入口，可独立配置额度；用户说“帮我画……”或携带/引用图片要求修改时，可在不影响其他生图插件的前提下调用本插件生图能力。
- 强化 QQ 空间评论收件箱去重：同时记录平台评论 ID、旧版退化 ID 和稳定评论指纹；拉取到评论后先落盘为已见，回复成功后立即落盘为已回复，降低 5 分钟轮询重复回复风险。

- 优化技能成长管理：等级展示改为“能力状态”口径，新增按分类折叠、隐藏、冻结成长和合并别名；隐藏技能仍可在页面管理，冻结技能保留能力边界但不再自动增长。
- 收敛技能成长注入：默认不再把完整技能列表塞进被动回复；用户明确提到技能名或合并别名时才按需注入相关能力状态，日程能力边界也只按技能名/别名触发，避免泛关键词误命中。
- 改进自动技能目录：细化学生、生活、创作、信息整理、冒险等分类，并温和迁移旧的泛分类，不覆盖用户手动设置的具体分类。
- 新增排障中心“技能相似项”检查：先本地筛出疑似重复、别名冲突和关键词高重叠项，再按需调用模型复核，只给建议不自动修改数据。
- 新增排障检查专用模型 `TROUBLESHOOTING_PROVIDER_ID`：排障中心模型复核优先使用该 Provider，留空时再回退到主动消息润色、陪伴通用和主模型，避免日常改写模型带偏诊断。
- 优化智能文本收口：`知道吗/问你个事/你猜/你懂吗` 这类短引子不再被“完整疑问句”快判直接放行，会先短等补话；智能收口小模型留空时明确回退到插件主模型。
- 优化引用媒体收口：单独发送图片或合并转发仍按图片/转发等待补充说明；引用对应图片或合并转发且附带文字时不再进入文本、群聊短唤醒或高强度收口等待。
- 修复合并转发等待后的二次等待：合并消息图片视觉/转述耗时较长时，如果收口缓冲早已到期，进入主链前会立即消费，不再额外等待一轮文本收口保护时间。
- 优化引用合并消息链路：引用合并消息已成功注入合并上下文时，不再把同一条引用里的图片重复走“引用图片”视觉链路，并复用本轮已解析的合并消息描述符。
- 优化图片补话收口：单图后马上补文字时，图片缓冲按首条图片的固定截止时间消费，不再因补话滑动延长；进入主链前会给后台视觉转述更合理的等待窗口，减少只注入路径导致“看不清”的情况。
- 补齐技能成长配置保存链路：静态配置区加入“被动回复技能认知”，确认 schema、运行时加载、WebUI、API 白名单和布尔转换一致。
