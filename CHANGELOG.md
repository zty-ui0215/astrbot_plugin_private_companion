# 更新日志

## 4.0.11

- 更新版本号至 `4.0.11`。
- 优化技能成长管理：等级展示改为“能力状态”口径，新增按分类折叠、隐藏、冻结成长和合并别名；隐藏技能仍可在页面管理，冻结技能保留能力边界但不再自动增长。
- 收敛技能成长注入：默认不再把完整技能列表塞进被动回复；用户明确提到技能名或合并别名时才按需注入相关能力状态，日程能力边界也只按技能名/别名触发，避免泛关键词误命中。
- 改进自动技能目录：细化学生、生活、创作、信息整理、冒险等分类，并温和迁移旧的泛分类，不覆盖用户手动设置的具体分类。
- 新增排障中心“技能相似项”检查：先本地筛出疑似重复、别名冲突和关键词高重叠项，再按需调用模型复核，只给建议不自动修改数据。
- 补齐技能成长配置保存链路：静态配置区加入“被动回复技能认知”，确认 schema、运行时加载、WebUI、API 白名单和布尔转换一致。

## 4.0.10

- 更新版本号至 `4.0.10`。
- 优化分段发送链路：普通 LLM 回复在 `all_llm` 范围下不再因链里包含引用、图片、语音、At 等非文本组件而整条跳过；只拆 `Plain` 文本，非文本组件作为原子块按顺序发送且不计入文本分段数。
- 收紧混合消息分段行为：只有文本真正拆成多段时才拆出组件顺序补发；若只是内容清理或无需分段，会保持原混合消息链，避免图文消息被误拆成多条。
- 新增 `segmented_message.py` 共用组件分段 helper，并让普通 LLM 分段和私聊图片回复分段共用同一套“文本拆分、媒体原子保留”逻辑，减少后续链路漂移。
- 修复分段配置一致性：`segmented_proactive_threshold` 运行时上限与配置页统一为 `1024`；分段正则、分段词、清理规则和间隔参数补齐总开关显示条件。

## 4.0.9

- 更新版本号至 `4.0.9`。
- 收敛拟人身体状态被动注入：完整和轻量路径统一为 `【当前扮演状态】`，只保留精力、情绪、粗粒度位置、必要睡眠/梦境/健康、饥饿、周期和额外叠加状态；时间、天气、日程和生活背景改由独立环境/生活片段负责。
- 优化固定状态文案：精力档位改为“完全没精神/提不起劲/状态一般/精神还不错/很精神”；睡眠、饥饿和周期池改为更自然的口语描述，并兼容旧值不误注入平稳项。
- 降噪状态注入内容：平稳周期、无饥饿感、睡得很踏实等基础状态不再写入当前扮演状态；第二行过滤睡眠、梦境、健康、饥饿、周期等已归类 condition，避免重复叠加。
- 同步扩展页和配置说明：生活观察页改用“当前扮演状态”口径，前端展示兼容旧值映射，配置页说明更新为“只影响语气、长短和节奏”。

## 4.0.8

- 更新版本号至 `4.0.8`。
- 重构 TTS 语音增强为两条生成路径：`fast_tag` 保留主模型私有标签的低延迟路径，`postprocess` 改为主模型纯文本、发送前由文本模型判断是否需要语音并完成翻译/改写；旧 `hybrid/direct/convert` 配置会自动兼容映射。
- 保留 TTS 既有发送层能力：TTS provider 合成、本机播放、直播字幕、语种修正、中文释义补全、分段补发、会话间隔、私聊/群聊概率覆盖和强/弱约束仍走统一发送前链路。
- 优化 TTS 配置页：后处理模式会隐藏快速标签自动语音、旧版概率和主用户强触发等无效配置项，切换生成路径或频率模式时立即刷新显示。
- 修复扩展页配置型子开关保存与回显链路：`settings` 型 bool 会与功能草稿合并显示，保存时自动拆回正确位置；后端也会兜底接收误发到 `features` 的 schema bool，降低“保存后退出回来又关闭”的概率。
- 统一扩展页百分比概率字段：`rest_reply_probability`、TTS 概率、群聊复读/唤醒概率等在页面中按 `0-100` 回显和保存，运行态再换算为 `0-1`；同时保留对 `0.18` 这类旧输入的兼容归一化。
- 调整后续维护目标：由于功能持续增加导致被动/主动/工具/TTS 等注入提示词逐渐臃肿，后续会优先推进注入提示词收敛、片段化组装、去重、优先级排序和污染控制；TTS 后续也会继续围绕提示词瘦身和路径稳定性整理。

## 4.0.7

- 修复朋友场景主动任务可能残留“偷看/屏幕观察”语义的问题：屏幕观察能力限定为主人授权下的本机屏幕，不再投射到朋友或远端用户。
- 优化主动候选、已计划任务和主动审计的 WebUI 展示：即使运行中存在旧候选数据，也会在展示前把朋友场景的敏感动作与动机清洗为普通文字关心。
- 同步更新屏幕观察能力说明与主动提示词示例，避免模型把“本机屏幕看一眼”误写成“偷看对方在干嘛”。
- 修复部分平台事件缺少明确 `group_id` 时可能把 `session_id`/发送者 QQ 误当群号的问题，避免白名单失效和扩展页群聊列表出现群成员 QQ。
- 修复扩展页用户详情保存每日主动上限时报 `_safe_int` 未定义的问题。
- 修复两处同类未定义名风险：群聊关系网自登记昵称归一化不再在静态方法里引用 `self`；“查看提示词 回复注入”会正确传入当前事件上下文。
- 对照 issue #17 复核 TTS 频率控制链路，并调整概率未命中日志文案，明确其作用是提示词层引导模型默认不输出 TTS，而不是硬剥离模型已输出的语音标签。
- 修复排障页最近注入内容看不到 TTS 的问题：TTS 请求规则会额外保留专用记录，并在排障页单独展示“TTS 规则注入”。
- 修复 direct/hybrid 下 LLM 主动输出 `<tts>` 可能被 AstrBot 原生 TTS 装饰器抢先处理的问题：模型响应阶段会先保护 TTS 块，发送前再交由插件统一执行语种修正、会话节流和可见中文释义补全。
- 新增 TTS 约束强度配置 `tts_constraint_mode`：默认弱约束保持“提示词引导 + LLM 判断”，强约束会在概率未命中或会话冷却内注入禁止语音的反向提示词，并在发送前阻止 `<pc_tts>/<tts>`、hybrid 自动语音和 convert 转换语音。
- 优化 LLM 自发语音标签链路：TTS 提示词改为推荐插件私有 `<pc_tts>` 标签，发送前再由插件统一处理，降低原生 `<tts>` 装饰器抢跑导致翻译/语种修正失效的概率；同时加硬概率未命中时的弱约束提示，没有用户明确语音请求时要求本轮必须纯文字。
- 新增休息模拟回复闸门：日程进入睡眠、午休或休息段后，可选按概率醒来回复，或由模型判断消息是否值得醒来；支持醒来阈值、概率和专用判断模型配置，默认关闭以保持旧行为。

## 4.0.6

- 修复 TTS 本机播放旧逻辑与配置文案不一致的问题：开启“生成后本机播放”后默认会播放所有来源的 TTS，只有额外开启“直播时仅播放直播回应消息”才限制为直播回应。
- 新增 TTS 本机播放直播限制开关 `enable_tts_local_playback_live_only`，并补齐扩展页显示、说明、保存白名单和运行时刷新。
- 优化直播 TTS 来源识别：支持直播插件传入的 `bili_live_auto_reply` 标记，也会按直播会话特征做兜底判断。
- 优化私聊轻量被动回复：短句轻量路径也会注入轻量环境感知，避免时间、平台和当前消息边界完全缺席；完整路径的环境感知也纳入模块化注入展示。
- 优化扩展页排障可读性：最近注入内容按模块展示每块用途，长内容折叠；消息收口防抖移到私聊页，排障页聚焦异常、链路测试、注入内容和 SQLite 并发。
- 优化排障页“最近问题”分类：普通主动调度过滤、休息中、冷却、低分和“不适合发送”等正常审计记录不再被当作异常展示，真正链路失败仍会保留。
- 新增排障页主动消息链路测试：点击后预约 1 分钟后的临时主动任务，由主动循环无视日限、冷却、活跃、免打扰等限制走完整主动生成流程，并分阶段展示目标会话、临时任务、LLM 渲染、时间复核、发送和历史归档结果。
- 修复午间/晚间独立主动问候仍可能承接旧邀约的问题：提示词和发送前检查都会识别“好呀、一直等着、想去哪儿逛”等回复式旧邀约痕迹，避免把历史聊天当成当前待办。
- 收敛主动页已登记任务展示：默认只显示关键任务，其余折叠，并减少后端返回量，降低页面拥挤和加载压力。
- 修复环境感知等内置整合能力可能被外部同类插件检测逻辑自动改回关闭的问题：启动时仅记录/提示重复注入风险，不再替用户改写开关配置。
- 修复扩展页功能开关与模块配置双入口回显不同步的问题：模块表单会优先显示当前功能开关草稿，避免环境感知这类开关在页面间切换时看起来又被关回去。
- 完善排障页最近注入检查：新增“请求附加规则”分组，展示 TTS 规则、TTS 频率控制、工具提示、环境感知、群聊人格降噪、能力边界、合并转发上下文等直接追加到请求的提示词。
- 收紧功能开关注入边界：新闻/私密阅读等总开关关闭后不再为了兜底说明继续注入对应上下文提示词。
- 优化被动注入的世界观适配：新增环境感知子开关 `enable_worldview_perception`，默认关闭；只有环境感知总开关和该子开关同时开启时才注入世界观适配片段，避免和 AstrBot 人设里已有世界观重复。
- 修复其他私聊用户主动发送失败难以排查的问题：手动启用的数字 QQ 用户会自动补齐私聊 UMO；主动发送失败日志会带出目标会话、平台、精确发送错误和核心回退错误，不再只显示空白失败。

## 4.0.4

- 优化 TTS 频率控制：全局概率作为提示词层面的方向约束，不再剥离 LLM 已输出的 `<tts>` 标签。
- 补充用户明确语音请求识别，例如“想听听你的声音”，让 LLM 在用户需要时可以顺应输出语音。
- 调整新版/旧版 TTS 提示词措辞：旧版更强调“适合采用”并由 LLM 判断，新版保留概率与会话间隔的整体节流。
- 新增私聊/群聊 TTS 概率与最小间隔覆盖项，默认 `-1` 继承全局，便于群聊低打扰、私聊更贴近陪伴。
- 新增默认关闭的“黑话释义联网参考”：可用 AstrBot 全局网页搜索为已有群黑话候选收集外部解释，并让模型判断与本群聊天样例的匹配程度。
- 修复扩展页配置保存链路：补齐新增 TTS 覆盖项和黑话联网参考配置的回显、白名单、类型归一化与运行时刷新，避免“面板能看到但保存不生效”。
- 修复后台整理任务出错后可能被新消息反复触发模型调用的问题：群聊片段、群黑话释义、私聊对话片段和长期画像在模型异常、JSON 无效或空摘要时会进入短冷却，并用运行中标记拦截同一用户/群的并发重复任务。
- 修复用户在上一条回复生成期间继续发消息时，旧回复的 TTS/分段后续块仍可能补发的问题：分段和 TTS 后台补发现在以触发消息的入站时间为基准，只要会话已有更新用户消息就停止旧尾巴。
- 增加主动消息发送前时间一致性复核：明显错时的主动文本会取消并重排，例如上午提前说放学、夜里接下午场景、问候时段与正文冲突等。

## 4.0.3

- 修复 4.0.1/4.0.2 日记生成后处理异常可能导致的重复 LLM 请求：日记生成成功后会先落库并标记当天已生成，梦境碎片合并失败只记录错误，不再触发下一轮重新生成。
- 增加日记生成失败冷却：真正生成失败时会进入 30 分钟冷却，避免主动循环反复调用模型。
- 增加后台维护失败通用冷却：日常状态、今日日程、当前细化、日记和创作推进等维护任务失败后会暂停一段时间再重试，避免同类后处理异常导致刷日志或重复请求。
- 补齐日记失败/后处理错误状态字段，便于后续排查启动和维护循环问题。
- 新增群聊复读阈值和计数选项：复读触发阈值默认 4、最小 3；可开启“复读只计不同用户”，避免一个人连续刷屏引发复读。
- 修复未配置主动生图后端时仍产生 `photo_text` 主动计划的问题：日程/细化规划会按真实后端可用性降级为普通文本，发送前也会把旧的 `photo_text` 计划降级为 `message`。
- 优化细化提示词：当主动生图不可用时明确禁止输出拍照/发图/[图片]类动作，避免计划文本和实际能力不一致。

## 4.0.2

- 修复引用图片/卡片的撤回阻断：发送前会同时检查当前触发消息和被引用原消息，引用图片、引用合并消息和引用卡片上下文在原消息撤回或不可见时不再继续注入。
- 优化私聊文本+图片混合消息：同一条私聊里附带图片/GIF 时由插件接入图片上下文，降低 AstrBot 本体图片上传失败后只按文字猜测的概率。

- 修复 Windows 下延迟保存偶发 `WinError 32`：数据保存改为唯一临时文件、原子替换重试和临时文件清理，减少文件短暂占用导致的保存失败。
- 引入 prompt 片段化组装入口：私聊被动状态注入先迁移到 `PromptSurface`/`PromptFragment`，为后续统一去重、优先级排序和污染控制打基础。

## 4.0.0

- 优化扩展页维护结构：拆分部分用户与群聊 API 逻辑，降低 `page_api.py` 的维护负担。
